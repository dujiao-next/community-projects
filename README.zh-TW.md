# Community Projects

[English](./README.md) | [简体中文](./README.zh-CN.md) | [繁體中文](./README.zh-TW.md)

> 更新時間：2026-02-17  
> 維護者：Dujiao-Next 社群

`community-projects` 用於集中收錄 Dujiao-Next 生態中的社群第三方貢獻內容，包含模板、腳本、工具與教學。
目標是讓貢獻者可以共享成果，讓使用者能快速發現並復用社群資源。

## 目錄說明

- `scripts/`：腳本類專案（部署腳本、遷移腳本、自動化腳本等）
- `templates/`：模板類專案（主題模板、頁面模板、郵件模板等）
- `tools/`：工具類專案（CLI、外掛、配套程式、視覺化工具等）
- `wikis/`：教學說明類內容（實作指南、最佳實踐、踩坑紀錄等）

## 提交範圍

歡迎提交以下內容：

1. 與 Dujiao-Next 生態相關的擴充專案
2. 可復用、可維護、可公開使用的開源專案
3. 對社群使用者有實際價值的模板、工具或教學

不建議提交以下內容：

1. 閉源且無明確使用說明的內容
2. 與 Dujiao-Next 無關的通用專案
3. 僅有占位檔案、無實際功能或無文件說明的內容

## 提交規範

每個專案請使用獨立目錄。命名建議：

- 使用小寫英文與連字號，例如：`payment-notify-helper`
- 避免空格、中文目錄名與特殊符號

建議結構：

```text
community-projects/
  templates/
    your-project-name/
      README.md
      LICENSE
      src/ 或 docs/ ...
```

## 必填要求

每個提交專案目錄下，至少包含以下資訊：

1. 開源協議
- 必須提供 `LICENSE` 檔案
- 建議使用 MIT / Apache-2.0 / GPL-3.0 等標準開源協議
- 若使用第三方資源，請在文件中說明授權來源

2. 使用教學
- 必須提供專案級 `README.md`
- 至少包含：專案簡介、安裝步驟、設定方式、使用示例、常見問題
- 若有環境相依（Node.js、Go、資料庫、系統版本等）請明確標示

3. 維護資訊
- 作者或維護者標識（暱稱/組織）
- 聯絡方式（選填，建議提供 issue 或討論區連結）
- 相容版本說明（適配的 Dujiao-Next 版本）

## README 建議章節

1. 專案介紹（解決什麼問題）
2. 功能列表
3. 安裝與部署
4. 設定說明
5. 使用教學（最小可執行示例）
6. 常見問題（FAQ）
7. 開源協議（License）
8. 更新紀錄（選填）

## 建議提交流程

1. Fork 並複製本倉庫
2. 在對應分類目錄下建立專案目錄
3. 補齊 `README.md` 與 `LICENSE`
4. 完成本地自測
5. 發起 Pull Request，並說明：
- 專案用途
- 適用場景
- 使用方式
- 相容版本

## 審核標準

維護者會重點檢查：

1. 文件完整度（尤其是開源協議與使用教學）
2. 專案是否可重現、可執行、可理解
3. 是否與 Dujiao-Next 生態相關
4. 是否存在侵權、敏感或不可公開散布內容

## 免責聲明

`community-projects` 下專案由社群貢獻者提供。
專案品質、穩定性與持續維護由貢獻者自行負責，使用前請自行評估並完成測試。
