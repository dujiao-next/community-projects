# Community Projects

[English](./README.md) | [简体中文](./README.zh-CN.md) | [繁體中文](./README.zh-TW.md)

> 更新时间：2026-02-17  
> 维护者：Dujiao-Next 社区

`community-projects` 用于集中收录 Dujiao-Next 生态的社区第三方贡献内容，包括模板、脚本、工具与教程。
目标是让贡献者能够共享成果，让使用者可以快速发现并复用社区资源。

## 目录说明

- `scripts/`：脚本类项目（部署脚本、迁移脚本、自动化脚本等）
- `templates/`：模板类项目（主题模板、页面模板、邮件模板等）
- `tools/`：工具类项目（CLI、插件、配套程序、可视化工具等）
- `wikis/`：教程说明类内容（实践指南、最佳实践、踩坑记录等）

## 提交范围

欢迎提交以下内容：

1. 与 Dujiao-Next 生态相关的扩展项目
2. 可复用、可维护、可公开使用的开源项目
3. 对社区用户有实际价值的模板、工具或教程

不建议提交以下内容：

1. 闭源且无明确使用说明的内容
2. 与 Dujiao-Next 无关的通用项目
3. 仅有占位文件、无实际功能或无文档说明的内容

## 提交规范

每个项目请使用独立目录。命名建议：

- 使用小写英文与连字符，例如：`payment-notify-helper`
- 避免空格、中文目录名和特殊符号

推荐结构：

```text
community-projects/
  templates/
    your-project-name/
      README.md
      LICENSE
      src/ 或 docs/ ...
```

## 必填要求

每个提交项目目录下，至少包含以下信息：

1. 开源协议
- 必须提供 `LICENSE` 文件
- 推荐使用 MIT / Apache-2.0 / GPL-3.0 等标准开源协议
- 若使用第三方资源，请在文档中说明授权来源

2. 使用教程
- 必须提供项目级 `README.md`
- 至少包含：项目简介、安装步骤、配置方法、使用示例、常见问题
- 如有环境依赖（Node.js、Go、数据库、系统版本等）请明确说明

3. 维护信息
- 作者或维护者标识（昵称/组织）
- 联系方式（可选，建议提供 issue 或讨论区链接）
- 兼容版本说明（适配的 Dujiao-Next 版本）

## README 建议章节

1. 项目介绍（解决什么问题）
2. 功能列表
3. 安装与部署
4. 配置说明
5. 使用教程（最小可运行示例）
6. 常见问题（FAQ）
7. 开源协议（License）
8. 更新记录（可选）

## 建议提交流程

1. Fork 并克隆仓库
2. 在对应分类目录下创建项目目录
3. 补齐 `README.md` 与 `LICENSE`
4. 完成本地自测
5. 发起 Pull Request，并说明：
- 项目用途
- 适用场景
- 使用方式
- 兼容版本

## 审核标准

维护者会重点检查：

1. 文档完整度（特别是开源协议与使用教程）
2. 项目是否可复现、可运行、可理解
3. 是否与 Dujiao-Next 生态相关
4. 是否存在侵权、敏感或不可公开分发内容

## 免责声明

`community-projects` 下项目由社区贡献者提供。
项目质量、稳定性与持续维护由贡献者自行负责，使用前请自行评估并做好测试。
